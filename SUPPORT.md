# Support

**Last reviewed:** 2026-04-13

This project is in alpha. Support is best-effort and focused on high-signal feedback.

## Which channel to use

- **Security vulnerability:** follow `SECURITY.md` (private disclosure only).
- **Bug report:** open a GitHub issue with `type:bug`.
- **Feature request:** open a GitHub issue with `type:feature`.
- **Docs/content improvement:** open a GitHub issue or PR with `type:docs`.
- **General question:** use GitHub Discussions when enabled; otherwise open an issue with `type:question`.

## Response expectations

- Critical reports (`priority:p0`): target acknowledgement in 24 hours.
- High-priority reports (`priority:p1`): target acknowledgement in 2 business days.
- Other requests: target acknowledgement in 5 business days.

See `docs/community-launch-playbook.md` for detailed triage and escalation workflow.

## Alpha support boundaries

To prevent maintainer overload during launch, support is currently limited to:

- Repository content in this project (`README`, `docs/`, `schemas/`, `examples/`, `packages/engine/`)
- Reproducible issues that include clear steps, environment details, and expected behavior
- Feedback aligned with the documented alpha scope

Support does not currently include:

- Real-time/24x7 support
- Custom consulting or private onboarding
- Troubleshooting third-party integrations without reproducible artifacts

## How to improve response quality

Include:

- environment (OS, Node.js version)
- exact commands run
- logs/error output
- minimal reproduction steps

Reports with reproducible detail are prioritized over broad or ambiguous requests.
