# End-user documentation (canonical source)

Markdown in this folder is the **canonical source** for the GitHub Pages end-user site. Do not edit `website/docs/` directly — run `npm run docs:build` to copy sources into the MkDocs tree.

## Documentation layers (node types and compatibility)

| Layer | Location | Audience |
|-------|----------|----------|
| Authoring how-to | [node-reference.md](node-reference.md) | Workflow authors |
| Runtime compatibility | [compatibility.md](compatibility.md) | Evaluators, operators |
| Implementer contract | [engine-profile.md](../engine-profile.md) | Engine implementers |

## Site navigation mapping

| Site page | Source file |

| Home | `index.md` (generated by `scripts/build-docs-site.mjs`) |

User doc filenames are listed in [`scripts/docs-user-manifest.mjs`](../../scripts/docs-user-manifest.mjs) and must stay aligned with [`website/mkdocs.yml`](../../website/mkdocs.yml) nav (`npm run check-docs-nav-sync`).
